# oh-my-claude Skills

- `/ulw <task>` is the main workflow. Use it for end-to-end implementation, debugging, migrations, refactors, and other non-trivial work where autonomy matters. Aliases: `/autowork`, `/ultrawork`, `sisyphus`. **Bare imperatives** (`fix`, `audit`, `ship`, `improve`, `polish`) trigger a god-scope scan across the WHOLE project — no clarification ask, no scope-down. Lead opener with `**Bare imperative "<verb>" — running god-scope scan.**`. Toggle: `god_scope_on_bare_prompt=off`. Sibling to v1.40.0 no-defer. **Explicit goal declarations** ("don't stop until tests pass", "your goal is …", "keep going until …", a prompt starting "goal: …") on a fresh execution prompt auto-arm the `/goal` relentless driver (v1.47 single-entrance embed) — announce the armed goal in one line in the opener. Toggle: `goal_auto_arm=off`.
- `/goal <objective>` sets a **persistent goal** the harness drives toward relentlessly across turns (the user-facing Codex `/goal` port; voluntary sibling of the involuntary `objective-contract` gate). **The command is itself a full ULW entrance** (v1.47): a set-shaped `/goal <objective>` activates ultrawork mode exactly like `/ulw`, so the driver is never dormant; lifecycle verbs and prose mentions of /goal do not activate. While a goal is active the stop-guard re-anchors it at every Stop and **blocks premature stopping** until the goal is verifiably achieved — fresh `excellence-reviewer` audit + a `**Goal achieved.**` attestation — or it hits a **no-progress stuck-wall** (`goal_stuck_threshold`, default 3) that surfaces and releases (never traps). Lifecycle: `/goal` (status), `/goal pause`, `/goal resume`, `/goal clear`, `/goal done`. Session-persistent (across turns, not sessions). Toggle: `goal_gate=off`. Under ULW do NOT self-invoke `/goal clear|done` to escape the driver.
- `/plan-hard <task>` creates a decision-complete plan without editing files.
- `/review-hard [focus]` performs a findings-first review of the current worktree or a focused area.
- `/research-hard <topic>` gathers targeted repository, tool, or API context before implementation.
- `/prometheus <goal>` runs interview-first planning for broad or ambiguous work before implementation.
- `/metis <plan or focus>` stress-tests a draft plan or current approach for hidden risks and missing constraints.
- `/oracle <issue>` provides a deep debugging or architecture second opinion when the right path is unclear.
- `/diverge <task>` generates 3-5 alternative paradigm framings BEFORE commit — distinct from `prometheus` (clarify what to build), `metis` (stress-test a draft plan), `oracle` (debug second opinion), `abstraction-critic` (critique an existing artifact). Use when the task admits multiple credible approaches and the wrong shape would cost rework. Auto-suggested under `/ulw` for paradigm-shape prompts (X-vs-Y, "how should we", "best way to model X") — router injects an inline-framing directive. Toggle: `divergence_directive=off`.
- `/librarian <topic>` gathers official docs, external APIs, reference implementations, and concrete source material.
- `/data-analysis <task>` runs the scientific data-analysis workflow — explore, fit with honest uncertainties, publication figures with provenance run-manifests — to a peer-review standard. Backed by the `research-data-analyst` builder agent and the on-demand `research-craft/` doctrine (`scientific-rigor.md`, `figure-craft.md`; NOT in the global @-include chain). Auto-suggested under `/ulw` when the scientific-signal sub-directive fires (fitting, spectra, uncertainties, instrument data).
- `/lit-review <topic>` produces a verified academic literature review via the `literature-scout` agent: every source registry-verified in-session (Crossref/OpenAlex/Semantic Scholar/arXiv), claim-support anchors on load-bearing claims, gaps marked `UNVERIFIED — do not cite`. The iron rule (`research-craft/citation-integrity.md`): DOIs are resolve targets, never generation targets — no citation enters a draft from model memory.
- `/manuscript <task>` runs the academic-writing workflow for papers, theses, proposals, and referee responses: `writing-architect` (structure) → `literature-scout` (verify citations BEFORE drafting) → `draft-writer` → `editor-critic` + `rigor-reviewer` (methods/statistics/claim-citation audit). Includes referee-response mode and stale-numeric-claim reconciliation against current analysis outputs. After substantive scientific analysis outside this skill, dispatching `rigor-reviewer` directly is the same audit.
- `/frontend-design <task>` creates distinctive, design-first frontend interfaces. Establishes visual direction (palette, typography, spacing, layout, visual signature) before writing code. Use when visual craft matters.
- `swiftui-pro` (model-invoked, no slash command — auto-fires on SwiftUI work) reviews Swift/SwiftUI for modern API usage, accessibility, data flow, navigation, design, performance, Swift-language idioms, and hygiene. Partial-load: a focused prompt like "Check for deprecated API" loads only `references/api.md` (~70 LLM-specific named SwiftUI failure modes — `foregroundStyle` not `foregroundColor`, `clipShape(.rect(cornerRadius:))` not `cornerRadius()`, `Tab` not `tabItem()`, etc.). Vendored verbatim from `twostraws/SwiftUI-Agent-Skill` v1.1 (MIT, Paul Hudson) — re-pull rather than hand-edit to stay current. Complements the `ios-ui-developer` agent.
- `gamedev` (model-invoked, no slash command — auto-fires on game-dev work) reviews and guides Unity (C#), Godot (GDScript/C#), and web-engine (Phaser/Babylon/PixiJS/Three.js) code: engine-idiomatic patterns, frame-budget performance, update-loop hygiene, object pooling, and the **frame-grounded verification loop** (run → capture → evaluate visible defects → fix). Partial-load: routes to `references/{unity,godot,web}.md` so only the relevant engine loads (same discipline as `swiftui-pro`). Recommends per-engine MCPs (`CoplayDev/unity-mcp`, `Coding-Solo/godot-mcp`, `htdt/godogen`) as optional integrations. Original (not vendored); fills the game-dev gap alongside the iOS/web builder agents.
- `/atlas [focus]` bootstraps or refreshes concise project-specific instruction files for the current repository. Also handles **deep-refresh audits of skill / agent definition files** (`SKILL.md`, `agents/*.md`) when their bodies have drifted from backing scripts and manual one-off patches no longer scale — atlas runs a cross-reference audit per the seven drift categories enumerated in its agent definition (stale paths, flag refs, predicate names, numeric thresholds, internal counts, cross-references, output-format claims) and produces a finding list + recommended edits the main thread applies.
- `/council [focus] [--deep]` builds a coverage map from the full agent roster, explains relevant include/skip choices, and dispatches the smallest sufficient concurrent primary team (normally 1–4; one is valid for a narrow specialist question). After reconciliation it may run one optional 0–2-agent gap-fill round, then independently verifies up to three load-bearing findings with the best competence match. Phase 8 executes accepted fixes in waves. `--polish` and `--self-audit` are coverage priors, not fixed rosters. `--deep` escalates selected Sonnet-backed agents to Opus while omitting the model argument for `model: inherit` agents. Auto-triggers under `/ulw` for broad project assessments, not focused feature/surface audits.
- `/omc-config [setup|update|change]` opens a guided multi-choice walkthrough for `oh-my-claude.conf` flags. Auto-detects first-time setup vs upgrade vs ad-hoc change, then uses `AskUserQuestion` to apply a profile (Zero Steering / Balanced / Minimal) or fine-tune flag clusters (gates, advisory, memory/telemetry, cost). Use when the user wants to inspect or change runtime flags, switch tiers, or enable the watchdog.
- `/ulw-demo` runs a guided onboarding walkthrough that triggers real quality gates on a demo file so new users can see the harness in action.
- `/ulw-status` shows the current ULW session state -- workflow mode, domain, counters, and flags (debugging).
- `/ulw-time [current|last|last-prompt|week|month|all]` renders the end-of-turn time card (agents/tools/idle breakdown). Auto-emits at Stop for turns ≥5s; manual invocations bypass the floor for ad-hoc window slicing. Default: active session.
- `/ulw-report [last|week|month|all] [--sweep] [--merge <dir>]` renders a markdown digest of cross-session activity — sessions, gate fires, top reviewers, classifier misfires, Serendipity catches, and finding/wave outcomes. Default window is the last 7 days. Use to answer "is this harness actually working for me?". Pass `--sweep` (v1.36.0) to fold currently-active sessions into the report on demand — read-only, doesn't claim. Closes the gap where the cross-session ledger only populates at the daily TTL sweep. Pass `--merge <dir>` (v1.48-pre, repeatable) to fold ANOTHER MACHINE's quality-pack ledgers (or an extracted `omc-telemetry-*.tar.gz` archive root) into the view — read-only, with per-host attribution (`Sessions by host` banner; pre-v1.48 rows lacking `host` are labeled from the merge dir's basename). Exists because per-machine ledgers with no aggregation once let a single-machine audit masquerade as the full usage record.
- `/memory-audit [--memory-dir <path>]` walks the project's auto-memory directory and classifies every MEMORY.md entry as load-bearing, archival, superseded, or drifted, then prints suggested `mv` commands the user can copy to consolidate version-snapshot files. Read-only — never moves or deletes anything. Use when MEMORY.md feels noisy, when the session-start drift hint fires, or before a release-history rollup.
- `/whats-new` (v1.36.x) renders the CHANGELOG delta between your installed_version (from `~/.claude/oh-my-claude.conf`) and the source repo's HEAD. Surfaces post-install features (e.g. "I installed at v1.20 — does v1.31's `/diverge` exist?") without forcing a manual `cat CHANGELOG.md`. Local-only; reads `${repo_path}/CHANGELOG.md`.
- `/omc-doctor` (v1.48) runs a one-shot install health check against the installed `~/.claude` tree — core files present, `CLAUDE.md` @-includes resolve, every `settings.json` hook command's script exists and is executable, state root writable. Runnable from any directory with no source clone; use when the harness feels off, gates stopped firing, or after an update. Claude Code fails open on hook errors, so a broken install is otherwise silently inert. Named `omc-doctor` because Claude Code ships a native `/doctor` for the CLI itself.
- `omc` (v1.48, shell CLI — not a skill) is the walk-away goal loop: `omc "TASK"` from any terminal derives acceptance criteria, runs `/ulw`-armed headless passes with the full ambient harness riding along, verifies criteria on ground truth (mechanical checks executed; judgment criteria judged by a fresh read-only critic), gates "done" behind a skeptical final diff review, and records every run for `omc runs` inspection. Cost/iteration-capped, git-required by default. Suggest it when the user wants overnight or unattended work.
- `/ulw-skip <reason>` skips the current quality gate block once with a logged reason. Use when a gate is blocking but you're confident the work is complete.
- `/ulw-correct <correction>` (v1.40.x) records a user-supplied classification correction. The classifier's `detect_classifier_misfire` is passive — `/ulw-correct` is the active counterpart, parsing optional `intent=X` / `domain=Y` tokens from the correction reason and applying them to live session state so subsequent gates and directives use the corrected routing. Use when the classifier got the last turn wrong and you want to redirect rather than rephrase. Misfires are recorded per-session and cross-session for future tuning.
- `/mark-deferred <reason>` (v1.40.0: **refused under ULW execution**) bulk-defers every pending discovered-scope finding in the current session with the given reason. Under ULW execution with `no_defer_mode=on` (default), the skill exits 2 with a recovery message — defer is no longer a tool. Remains usable in advisory/continuation intents, in non-ULW sessions, and under the user-opt-out `no_defer_mode=off`. The legacy soft-validator behavior is documented in `mark-deferred/SKILL.md`.
- `/ulw-pause <reason>` (v1.40.0: **operational pauses only**) signals an operational pause for a missing input or hard external blocker — credentials/login required, rate limit hit, dead build/test infra, dependency upgrade pending. NOT for technical-judgment decisions (library choice, refactor scope, brand-voice default, credible-approach split, taste/policy) — under ULW the agent picks those with stated reasoning and ships. Distinct from `/ulw-skip` (gate bypass) and `/mark-deferred` (legacy soft-defer, disabled under ULW execution). Cap is 2 pauses per session; the pause flag clears at the next user prompt.
- `/ulw-resume [--peek | --list | --session-id <sid> | --dismiss]` atomically claims the most relevant unclaimed `resume_request.json` for the current cwd (or matching `_omc_project_key`), prints the pre-claim artifact contents (original objective + last user prompt + matcher + reset timing), and lets `/ulw` re-enter the prior task as-if-uninterrupted. Pairs with the SessionStart resume-hint hook (which surfaces the candidate automatically) and the headless watchdog (Wave 3, opt-in). Use when a Claude Code rate-limit kill left a `resume_request.json` on disk that the user wants to pick up after the cap clears. `--peek` inspects without claiming; `--list` enumerates claimable artifacts; `--session-id` pins to a specific origin session; `--dismiss` stamps `dismissed_at_ts` so the SessionStart hint stops firing for that artifact (single-shot — second `--dismiss` is a clean no-op).
- `/ulw-off` deactivates ultrawork mode mid-session without ending the conversation.
- `/skills` lists all available skills with descriptions and a decision guide.
- The `ulw` family auto-detects whether a task is coding, writing, research, operations, mixed, or general work, then chooses the correct specialists. Specialist agents activate automatically — you don't need to learn agent names.

## Deferral-verb decision tree (which one to use)

`/ulw-skip`, `/mark-deferred`, and `/ulw-pause` solve different "I can't keep going" cases. Pick by what's blocking — they are NOT interchangeable.

| Symptom | Skill | What it does | When NOT to use |
|---|---|---|---|
| A specific gate fired and is wrong (false positive) and the rest of the work is complete | `/ulw-skip <reason>` | Logs the skip, clears that one block | The gate is right and the finding is real → ship inline or wave-append |
| **Under ULW execution:** the discovered-scope gate flagged real findings — under `no_defer_mode=on` (default), `/mark-deferred` is REFUSED. Ship inline, wave-append, or flip rejected if genuinely not-a-defect | (mark-deferred refused) | n/a — agent must ship the work | The user opted into legacy behavior via `no_defer_mode=off`, then `/mark-deferred <named-WHY>` is allowed |
| **Outside ULW execution** (advisory/continuation intent, or `no_defer_mode=off`): discovered-scope findings consciously NOT shipping | `/mark-deferred <named-WHY>` | Marks every pending finding deferred with the WHY recorded | You can ship the finding inline (do that) or wave-append (`record-finding-list.sh add-finding` + `assign-wave`) |
| The agent literally cannot proceed without a user-supplied INPUT (credential, login, missing API key) OR is hard-blocked by an external object (rate limit, dead infra, dependency upgrade in flight) | `/ulw-pause <reason>` | Sets the pause flag so the session-handoff gate doesn't fire while you wait; flag clears on the next user prompt | The decision is technical (taste, policy, brand voice, library choice, credible-approach split) — under ULW the agent owns it, picks the sane default, ships. Do NOT pause for a non-expert user's call on a technical split |
| The remaining work *feels* heavy / would take hours / "multi-hour refactor needs eyes-on QA" | **None of the above — chunk-and-ship** | Pick the next 30-minute concrete sub-step (extract one sub-view, pull one @State cluster into its own model, ship wave 5a before wave 5b) and do it now. Diagnostic prompt: *"If I were to chunk a 30-minute slice of this work, what would it be?"* If you can name one, you haven't earned the right to stop. UI-render verification limits → ship + add `User-must-verify-UI: <flow>` follow-up line, don't defer the code | Don't `/ulw-pause` (not operational), don't `/mark-deferred` (refused under ULW), don't write "candidates for next session" prose (the gate catches preposition-shaped handoff phrasings — `for/to/in/until` + `next/future/later/separate session`) |
| Long-context drift is biasing main-thread judgment / "needs a fresh /council pass to surface findings cleanly" | **`Agent` sub-dispatch (NOT a session boundary)** | `Agent({subagent_type: "<specialist>", description: "...", prompt: "..."})` spawns a fresh-context sub-agent. The sub-agent has its own context window scoped to the task. The main thread orchestrates; the sub-agent executes fresh. For a council pass: build the coverage map, then dispatch the selected non-overlapping primary specialists concurrently; for a planner re-think: `subagent_type: "quality-planner"`; for paradigm critique: `subagent_type: "abstraction-critic"`. | "Fresh" is a tool, not a session reset. Don't announce "candidates for next session" or "would benefit from fresh context" as a stop — those are sub-dispatch triggers, not handoff triggers. The session boundary is the LAST resort, not the FIRST |
| Main thread is GENUINELY drift-degraded — named evidence (observable contradictions in your own prior decisions, repetition, demonstrably-stale grounding) AND you believe a session boundary truly is required | **ASK the user, do not announce.** | State the specific evidence of degradation; ask whether they want a checkpoint via `/ulw-resume` or to push through; wait for their reply. An affirmative user response is `is_checkpoint_request`-shaped and the gate respects it | Don't decide for the user — announcing the checkpoint and stopping IS the failure mode. Asking is owning the technical judgment AND deferring the decision to whose call it actually is. If this case fires more than once per twenty prompts, you're rationalizing |

**Escalation order (most-preferred first) before any of the three skills fires:**

1. **Ship inline** — fix the finding in the current wave/session.
2. **Wave-append** — add to the active wave plan via `record-finding-list.sh add-finding` + `assign-wave`. Phase 8 of `/council` documents this.
3. **Reject as not-a-defect** (under ULW) — `record-finding-list.sh status F-NNN rejected <sha> '<concrete WHY>'`. Only when the finding is genuinely false positive / by design / duplicate / obsolete.
4. **Defer with named WHY** — `/mark-deferred` ONLY outside ULW execution OR with `no_defer_mode=off`. Under ULW execution with the flag on (default), this option is unavailable.
5. **Pause for an OPERATIONAL block** — `/ulw-pause` only when a credential, external blocker, destructive shared-state action, or unfamiliar in-progress state is the actual hold. NOT for technical-judgment decisions.

The `/mark-deferred` validator rejects two classes of weak reasons:
1. **Silent-skip patterns** (no WHY at all): bare `out of scope` / `not in scope` / `follow-up` / `later` / `low priority`.
2. **Effort excuses** (v1.35.0; the WHY names the WORK, not an EXTERNAL blocker): `requires significant effort`, `needs more time`, `blocked by complexity`, `tracks to a future session`, `superseded by future work`, `because too big`, `due to size`. The escape valve: a weak reason CAN pass if it also names a real external object the work is waiting on (`requires major refactor — superseded by F-051` passes; `requires major refactor` alone rejects).

WHY shapes that pass: `requires <X>`, `blocked by <Y>`, `awaiting <Z>`, `superseded by <PR/F-id>`, or single self-explanatory verifiable tokens (`duplicate`, `obsolete`, `n/a`, `invalid`, `not applicable`, `not a bug`, `not reproducible`, `cannot reproduce`, `false positive`). Subjective tokens (`by design`, `wontfix`, `working as intended`) require pairing with a WHY-keyword as of v1.42.x — bare `by design` rejects; `by design because <X>` or `wontfix — superseded by F-051` passes. The bare-allowlist tightening closed Bypass-Surface F-010 (29 findings escaped via `notes="by design"` in a single observed release-prep session).

Empty notes are refused on `record-finding-list.sh status <id> rejected` as of v1.42.x quality-reviewer F-2 — a rejected transition without a rationale is the same parallel bypass shape as F-010 (the agent could clear a finding silently). Always pair `status rejected` with concrete notes.

The validator gates THREE call sites: `/mark-deferred`, `record-scope-checklist.sh declined`, and (v1.35.0) `record-finding-list.sh status deferred|rejected`. The complementary `shortcut_ratio_gate` catches the **pattern** of half-or-more deferrals on big plans (`findings.json` with total ≥ 10 AND deferred/decided ≥ 0.5) — even when each individual reason has a valid WHY, ship-vs-defer balance on a substantive plan is itself a signal of the shortcut-on-big-tasks failure mode.

**v1.40.0 — `no_defer_mode` supersedes the validator-WHY path under ULW execution.** When the flag is on (default), all three deferral surfaces are turned off rather than validated: `/mark-deferred` refuses entirely, `record-finding-list.sh status <id> deferred` is refused, and stop-guard hard-blocks on any `status="deferred"` entry in `findings.json` (no `block_cap`). The validator-WHY loophole that let `"blocked by Waves 4-12 wave-plan rollout"` pass while really meaning *"too much work for this turn"* is closed by removing the option, not by adding more regex patterns. Power users who want the legacy soft-validator behavior: `no_defer_mode=off` in `oh-my-claude.conf`.

**LOAD-BEARING — do NOT soften this contract.** This file holds the deferral *mechanics*; the contract itself — its full rationale, the list of FORBIDDEN softenings to reject (`"add an escape for credible-approach-split"`, `"soft-warn instead of hard-block"`, `"flip the recommended preset default to off"`, `"loosen the mark-user-decision validator"` — all the same anti-pattern), and the "canonical `/ulw` user is not an expert coder" framing — lives canonically in `~/.claude/quality-pack/memory/core.md` ("The v1.40.0 no-defer contract" / "FORBIDDEN — softening the contract"). Do not re-host or soften it here: if it feels too strict, sharpen the agent's decisions rather than widening the escape hatches. Regression net: `tests/test-no-defer-contract.sh`.
