# oh-my-claude Skills

- `/ulw <task>` is the main workflow. Use it for end-to-end implementation, debugging, migrations, refactors, and other non-trivial work where autonomy matters. Aliases: `/autowork`, `/ultrawork`, `sisyphus` also work.
- `/plan-hard <task>` creates a decision-complete plan without editing files.
- `/review-hard [focus]` performs a findings-first review of the current worktree or a focused area.
- `/research-hard <topic>` gathers targeted repository, tool, or API context before implementation.
- `/prometheus <goal>` runs interview-first planning for broad or ambiguous work before implementation.
- `/metis <plan or focus>` stress-tests a draft plan or current approach for hidden risks and missing constraints.
- `/oracle <issue>` provides a deep debugging or architecture second opinion when the right path is unclear.
- `/diverge <task>` (v1.31.0) generates 3-5 alternative paradigm framings BEFORE commit — distinct from `prometheus` (clarify what to build), `metis` (stress-test a draft plan), `oracle` (debug second opinion), `abstraction-critic` (critique an existing artifact). Use when the task admits multiple credible approaches and choosing the wrong shape would cost rework. **Auto-suggested under `/ulw` (v1.32.0):** when a prompt names a paradigm-shape decision (X-vs-Y choice, "how should we", "best way to model X", "design the X strategy", "is there a better way") the prompt-intent-router injects a `DIVERGENT-FRAMING DIRECTIVE` instructing the model to enumerate 2-3 framings INLINE in its opener — label + mental model + EASY/HARD affordance + redirect-if clause. Default is inline lateral thinking; explicit `/diverge` invocation remains the heavier escalation when inline enumeration feels shallow. Toggle off via `divergence_directive=off` (env `OMC_DIVERGENCE_DIRECTIVE=off`).
- `/librarian <topic>` gathers official docs, external APIs, reference implementations, and concrete source material.
- `/frontend-design <task>` creates distinctive, design-first frontend interfaces. Establishes visual direction (palette, typography, spacing, layout, visual signature) before writing code. Use when visual craft matters.
- `/atlas [focus]` bootstraps or refreshes concise project-specific instruction files for the current repository. Also handles **deep-refresh audits of skill / agent definition files** (`SKILL.md`, `agents/*.md`) when their bodies have drifted from backing scripts and manual one-off patches no longer scale — atlas runs a cross-reference audit per the seven drift categories enumerated in its agent definition (stale paths, flag refs, predicate names, numeric thresholds, internal counts, cross-references, output-format claims) and produces a finding list + recommended edits the main thread applies.
- `/council [focus] [--deep]` dispatches a multi-role project-evaluation panel (PM, design, security, data, SRE, growth). Includes a Phase 7 verification pass that re-checks the top 2-3 findings via `oracle` before presenting, and a Phase 8 (Execution Plan) that bridges assessment to wave-based implementation when the prompt requests fixes — group findings into 5–10-finding waves by surface area, execute each wave fully (plan → impl → quality-reviewer on the wave's diff → excellence-reviewer → verify → per-wave commit titled `Wave N/M: <surface>`) before the next, track status in `<session>/findings.json` via `record-finding-list.sh`. The five-priority rule governs presentation order, not execution scope. `--deep` escalates lenses to opus for higher-stakes audits (cost: meaningfully higher per run). Also auto-triggers under `/ulw` when the prompt asks for a broad project assessment.
- `/omc-config [setup|update|change]` opens a guided multi-choice walkthrough for `oh-my-claude.conf` flags. Auto-detects first-time setup vs upgrade vs ad-hoc change by reading the conf, then uses `AskUserQuestion` to apply a profile (Zero Steering / Balanced / Minimal) or fine-tune individual flag clusters (gates, advisory, memory/telemetry, cost). The legacy `maximum` token remains a valid alias for Zero Steering. When the user opts into `resume_watchdog=on` or a different `model_tier`, the skill chains to `install-resume-watchdog.sh` / `switch-tier.sh` for the side effects. Use when the user says "help me install/update/configure oh-my-claude", "show my settings", "switch to max-automation", "turn the watchdog on", or any phrase asking to inspect or change runtime flags.
- `/ulw-demo` runs a guided onboarding walkthrough that triggers real quality gates on a demo file so new users can see the harness in action.
- `/ulw-status` shows the current ULW session state -- workflow mode, domain, counters, and flags (debugging).
- `/ulw-time [current|last|last-prompt|week|month|all]` renders a polished end-of-turn time card — stacked top bar (`█` agents · `▒` tools · `░` idle), per-bucket ASCII chart with per-row bars and call counts, and a one-line insight (anomaly outranks dominance outranks reassurance outranks fun fact). The same card auto-emits as Stop `additionalContext` at the end of every turn whose walltime is `>= 5 seconds`; below the floor the hook stays silent. Manual invocations bypass the floor — useful when you want to slice a different window without waiting for a Stop. Default mode is the active session.
- `/ulw-report [last|week|month|all] [--sweep]` renders a markdown digest of cross-session activity — sessions, gate fires, top reviewers, classifier misfires, Serendipity catches, and finding/wave outcomes. Default window is the last 7 days. Use to answer "is this harness actually working for me?". Pass `--sweep` (v1.36.0) to fold currently-active sessions into the report on demand — read-only, doesn't claim. Closes the gap where the cross-session ledger only populates at the daily TTL sweep.
- `/memory-audit [--memory-dir <path>]` walks the project's auto-memory directory and classifies every MEMORY.md entry as load-bearing, archival, superseded, or drifted, then prints suggested `mv` commands the user can copy to consolidate version-snapshot files. Read-only — never moves or deletes anything. Use when MEMORY.md feels noisy, when the session-start drift hint fires, or before a release-history rollup.
- `/whats-new` (v1.36.x) renders the CHANGELOG delta between your installed_version (from `~/.claude/oh-my-claude.conf`) and the source repo's HEAD. Surfaces post-install features (e.g. "I installed at v1.20 — does v1.31's `/diverge` exist?") without forcing a manual `cat CHANGELOG.md`. Local-only; reads `${repo_path}/CHANGELOG.md`.
- `/ulw-skip <reason>` skips the current quality gate block once with a logged reason. Use when a gate is blocking but you're confident the work is complete.
- `/ulw-correct <correction>` (v1.40.x) records a user-supplied classification correction. The classifier's `detect_classifier_misfire` is passive — `/ulw-correct` is the active counterpart, parsing optional `intent=X` / `domain=Y` tokens from the correction reason and applying them to live session state so subsequent gates and directives use the corrected routing. Use when the classifier got the last turn wrong and you want to redirect rather than rephrase. Misfires are recorded per-session and cross-session for future tuning.
- `/mark-deferred <reason>` (v1.40.0: **refused under ULW execution**) bulk-defers every pending discovered-scope finding in the current session with the given reason. Under ULW execution with `no_defer_mode=on` (default), the skill exits 2 with a recovery message — defer is no longer a tool. Remains usable in advisory/continuation intents, in non-ULW sessions, and under the user-opt-out `no_defer_mode=off`. The legacy soft-validator behavior is documented in `mark-deferred/SKILL.md`.
- `/ulw-pause <reason>` (v1.40.0: **operational pauses only**) signals an operational pause for a missing input or hard external blocker — credentials/login required, rate limit hit, dead build/test infra, dependency upgrade pending. NOT for technical-judgment decisions (library choice, refactor scope, brand-voice default, credible-approach split, taste/policy) — under ULW the agent picks those with stated reasoning and ships. Distinct from `/ulw-skip` (gate bypass) and `/mark-deferred` (legacy soft-defer, disabled under ULW execution). Cap is 2 pauses per session; the pause flag clears at the next user prompt.
- `/ulw-resume [--peek | --list | --session-id <sid> | --dismiss]` atomically claims the most relevant unclaimed `resume_request.json` for the current cwd (or matching `_omc_project_key`), prints the pre-claim artifact contents (original objective + last user prompt + matcher + reset timing), and lets `/ulw` re-enter the prior task as-if-uninterrupted. Pairs with the SessionStart resume-hint hook (which surfaces the candidate automatically) and the headless watchdog (Wave 3, opt-in). Use when a Claude Code rate-limit kill left a `resume_request.json` on disk that the user wants to pick up after the cap clears. `--peek` inspects without claiming; `--list` enumerates claimable artifacts; `--session-id` pins to a specific origin session; `--dismiss` stamps `dismissed_at_ts` so the SessionStart hint stops firing for that artifact (single-shot — second `--dismiss` is a clean no-op).
- `/ulw-off` deactivates ultrawork mode mid-session without ending the conversation.
- `/skills` lists all available skills with descriptions and a decision guide.
- The `ulw` family auto-detects whether a task is coding, writing, research, operations, mixed, or general work, then chooses the correct specialists. Specialist agents activate automatically — you don't need to learn agent names.

## Deferral-verb decision tree (which one to use)

`/ulw-skip`, `/mark-deferred`, and `/ulw-pause` solve different "I can't keep going" cases. Pick by what's blocking — they are NOT interchangeable.

| Symptom | Skill | What it does | When NOT to use |
|---|---|---|---|
| A specific gate fired and is wrong (false positive) and the rest of the work is complete | `/ulw-skip <reason>` | Logs the skip, clears that one block | The gate is right and the finding is real → ship inline or wave-append |
| **Under ULW execution:** the discovered-scope gate flagged real findings — under `no_defer_mode=on` (default), `/mark-deferred` is REFUSED. Ship inline, wave-append, or flip rejected if genuinely not-a-defect | (mark-deferred refused) | n/a — agent must ship the work | The user opted into legacy behavior via `no_defer_mode=off`, then `/mark-deferred <named-WHY>` is allowed |
| **Outside ULW execution** (advisory/continuation intent, or `no_defer_mode=off`): discovered-scope findings consciously NOT shipping | `/mark-deferred <named-WHY>` | Marks every pending finding deferred with the WHY recorded | You can ship the finding inline (do that) or wave-append (`record-finding-list.sh add-finding` + `assign-wave`) |
| The agent literally cannot proceed without a user-supplied INPUT (credential, login, missing API key) OR is hard-blocked by an external object (rate limit, dead infra, dependency upgrade in flight) | `/ulw-pause <reason>` | Sets the pause flag so the session-handoff gate doesn't fire while you wait; flag clears on the next user prompt | The decision is technical (taste, policy, brand voice, library choice, credible-approach split) — under ULW the agent owns it, picks the sane default, ships. Do NOT pause for a non-expert user's call on a technical split |

**Escalation order (most-preferred first) before any of the three skills fires:**

1. **Ship inline** — fix the finding in the current wave/session.
2. **Wave-append** — add to the active wave plan via `record-finding-list.sh add-finding` + `assign-wave`. Phase 8 of `/council` documents this.
3. **Reject as not-a-defect** (under ULW) — `record-finding-list.sh status F-NNN rejected <sha> '<concrete WHY>'`. Only when the finding is genuinely false positive / by design / duplicate / obsolete.
4. **Defer with named WHY** — `/mark-deferred` ONLY outside ULW execution OR with `no_defer_mode=off`. Under ULW execution with the flag on (default), this option is unavailable.
5. **Pause for an OPERATIONAL block** — `/ulw-pause` only when a credential, external blocker, destructive shared-state action, or unfamiliar in-progress state is the actual hold. NOT for technical-judgment decisions.

The `/mark-deferred` validator rejects two classes of weak reasons:
1. **Silent-skip patterns** (no WHY at all): bare `out of scope` / `not in scope` / `follow-up` / `later` / `low priority`.
2. **Effort excuses** (v1.35.0; the WHY names the WORK, not an EXTERNAL blocker): `requires significant effort`, `needs more time`, `blocked by complexity`, `tracks to a future session`, `superseded by future work`, `because too big`, `due to size`. The escape valve: a weak reason CAN pass if it also names a real external object the work is waiting on (`requires major refactor — superseded by F-051` passes; `requires major refactor` alone rejects).

WHY shapes that pass: `requires <X>`, `blocked by <Y>`, `awaiting <Z>`, `superseded by <PR/F-id>`, or single self-explanatory tokens/phrases (`duplicate`, `obsolete`, `wontfix`, `n/a`, `not reproducible`, `false positive`, `by design`, `working as intended`).

The validator gates THREE call sites: `/mark-deferred`, `record-scope-checklist.sh declined`, and (v1.35.0) `record-finding-list.sh status deferred|rejected`. The complementary `shortcut_ratio_gate` catches the **pattern** of half-or-more deferrals on big plans (`findings.json` with total ≥ 10 AND deferred/decided ≥ 0.5) — even when each individual reason has a valid WHY, ship-vs-defer balance on a substantive plan is itself a signal of the shortcut-on-big-tasks failure mode.

**v1.40.0 — `no_defer_mode` supersedes the validator-WHY path under ULW execution.** When the flag is on (default), all three deferral surfaces are turned off rather than validated: `/mark-deferred` refuses entirely, `record-finding-list.sh status <id> deferred` is refused, and stop-guard hard-blocks on any `status="deferred"` entry in `findings.json` (no `block_cap`). The validator-WHY loophole that let `"blocked by Waves 4-12 wave-plan rollout"` pass while really meaning *"too much work for this turn"* is closed by removing the option, not by adding more regex patterns. Power users who want the legacy soft-validator behavior: `no_defer_mode=off` in `oh-my-claude.conf`.
