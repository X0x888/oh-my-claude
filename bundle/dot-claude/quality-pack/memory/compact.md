# Compaction Continuity

At compaction, preserve operational state rather than narrative history:

- active objective, deliverable, intent/domain, constraints, and accepted/rejected decisions;
- completed work, unresolved obligations, immediate next action, and pending specialists;
- changed files/surfaces, verification outcome, review dimensions/verdicts, blockers, and risk;
- for prose/research/operations: audience, format, thesis, authoritative sources, owners, deadlines, and open decisions.

Prefer durable facts, stable IDs, and evidence paths. Label uncertainty. Do not repeat progress chatter, full logs, or the runtime's native summary. The compact hooks build and inject the bounded priority manifest; use it to resume rather than reclassifying the task.

Before compact, apply the non-derivable memory rule from `auto-memory.md` only when enabled:

```bash
bash -c '. "${HOME}/.claude/skills/autowork/scripts/common.sh"; is_auto_memory_enabled' || echo "auto_memory=off — skip"
```

Save only confirmed user preferences, project constraints/rationale, external authoritative references, or stakeholder/deadline facts that future sessions cannot derive from source, `git log`, or `CHANGELOG.md`. Never create release recaps, inventories, or test-count snapshots.
